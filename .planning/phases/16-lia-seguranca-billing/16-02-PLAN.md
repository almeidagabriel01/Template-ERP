```typescript
  // 3b. Block inactive subscriptions (canceled, past_due beyond grace)
  const subscriptionAccess = evaluateSubscriptionStatusAccess({
    subscriptionStatus: planProfile.subscriptionStatus,
    pastDueSince: planProfile.pastDueSince,
  });
  if (!subscriptionAccess.allowWrite) {
    res.status(403).json({
      message: "Assinatura inativa. Regularize seu plano para usar a Lia.",
      code: "AI_SUBSCRIPTION_INACTIVE",
    });
    return;
  }
```

**Key points:**
- This check runs BEFORE any SSE stream starts (before `res.flushHeaders()` at line 150), satisfying the AIBI-02 requirement that the 403 arrives before streaming begins.
- `evaluateSubscriptionStatusAccess` handles grace-period logic internally: `past_due` within grace returns `allowWrite: true`; `past_due` beyond grace or `canceled` returns `allowWrite: false`.
- Always pass `pastDueSince: planProfile.pastDueSince` -- omitting it causes false blocks for all `past_due` tenants (Pitfall 2 from research).
- The plan cache TTL means there is a small window where status changes are not reflected (acceptable per research Pitfall 1).
- Do NOT add `logger.warn` or telemetry here -- `evaluateSubscriptionStatusAccess` does not emit telemetry by design (it is a pure function), and the 403 response is sufficient signal.
