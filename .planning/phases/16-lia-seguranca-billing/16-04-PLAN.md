**Step 2: Add dismiss state and computed flag inside the `LiaContainer` function body.**

After `const usage = useLiaUsage();` (line 90), add:

```typescript
  const [nearLimitDismissed, setNearLimitDismissed] = useState(false);
  const showNearLimitBanner = usage.isNearLimit && !usage.isAtLimit && !nearLimitDismissed;
```

**Key logic:** The banner only shows when `isNearLimit` is true AND `isAtLimit` is false AND not dismissed. When `isAtLimit` is true, the input bar is already disabled with a tooltip (AIBI-03) -- showing both a banner and disabled input would be redundant (research Pitfall 4).

**Step 3: Modify the `inputBar` slot in the LiaPanel JSX to prepend the banner.**

Replace the current `inputBar` slot (approximately lines 181-187):
```tsx
        inputBar={
          <LiaInputBar
            onSend={chat.sendMessage}
            isStreaming={chat.isStreaming}
            isAtLimit={usage.isAtLimit}
            resetDate={usage.resetDate}
          />
        }
```

with:
```tsx
        inputBar={
          <>
            {showNearLimitBanner && (
              <div className="flex items-center justify-between gap-2 px-4 py-2 bg-amber-50 dark:bg-amber-950/40 border-t border-amber-200 dark:border-amber-800 text-sm text-amber-700 dark:text-amber-300 shrink-0">
                <span>
                  Voc\u00ea usou {usage.messagesUsed} de {usage.messagesLimit} mensagens este m\u00eas. Renova em {usage.resetDate}.
                </span>
                <button
                  type="button"
                  onClick={() => setNearLimitDismissed(true)}
                  aria-label="Fechar aviso de limite"
                  className="shrink-0 p-0.5 rounded hover:bg-amber-200/50 dark:hover:bg-amber-800/50 transition-colors"
                >
                  <span aria-hidden="true" className="text-base leading-none">\u2715</span>
                </button>
              </div>
            )}
            <LiaInputBar
              onSend={chat.sendMessage}
              isStreaming={chat.isStreaming}
              isAtLimit={usage.isAtLimit}
              resetDate={usage.resetDate}
            />
          </>
        }
```

**Implementation details:**
- The banner uses `\u00ea` (e-circumflex for "Voce") and `\u00eas` (for "mes") and `\u2715` (multiplication X for close button) -- use the actual Unicode characters in the file: `Você usou ... mensagens este mês`.
- Amber color scheme matches the LiaUsageBadge amber state for visual consistency.
- `dark:bg-amber-950/40` and `dark:text-amber-300` provide dark mode support.
- `shrink-0` prevents the banner from being squeezed by flex layout.
- `border-t` creates a visual separator from the chat window above.
- The dismiss state (`nearLimitDismissed`) is session-only (useState) -- it resets on page reload, which is intentional: the warning should reappear on new sessions.
- The Fragment (`<>...</>`) wraps both banner and LiaInputBar to pass as a single ReactNode to the `inputBar` slot.
