# Plan 2: Lighthouse Performance + Security Scan Scripts

## Goal
Developer can run `npm run test:performance` to generate a Lighthouse report and `npm run test:security` to run a lightweight security scan locally, satisfying INFRA-02 and INFRA-03.

## Context
Per D-03: `npm run test:security` locally runs a lightweight JS-based security check (npm audit + header/CORS validation), NOT full OWASP ZAP. Full ZAP scan runs only in CI (covered in Plan 3). Per INFRA-02: `npm run test:performance` generates a Lighthouse report locally. This plan is independent of Plan 1 (no shared files) and can execute in parallel.

## Tasks

<task id="1.2.1" title="Create Lighthouse local runner and security scan scripts">
<read_first>
- package.json -- current scripts to avoid naming conflicts, current dependencies
- firebase.json -- emulator ports for security checks against local server
- .env.test -- test environment variables (created by Plan 1, but this task only needs to know the pattern)
</read_first>
<steps>
1. Install dev dependencies:
   - `@lhci/cli` -- Lighthouse CI CLI for running audits
   - `lighthouse` -- core library for programmatic access (if not included with @lhci/cli)

2. Create `e2e/lighthouse/lighthouse.config.ts`:
   - Export Lighthouse CI configuration object with:
     - `ci.collect.url`: `['http://localhost:3000/login']` (single page for local baseline; CI will add more)
     - `ci.collect.numberOfRuns`: 1 (local) -- CI overrides to 3
     - `ci.collect.settings.chromeFlags`: `['--headless', '--no-sandbox']`
     - `ci.collect.settings.onlyCategories`: `['performance', 'accessibility', 'best-practices']`
     - `ci.assert.assertions`: Core Web Vitals thresholds:
       - `first-contentful-paint`: `["warn", { maxNumericValue: 2500 }]`
       - `largest-contentful-paint`: `["error", { maxNumericValue: 4000 }]` (relaxed for local, CI tightens to 2500)
       - `cumulative-layout-shift`: `["error", { maxNumericValue: 0.1 }]`
       - `total-blocking-time`: `["warn", { maxNumericValue: 300 }]` (TBT as FID proxy)
     - `ci.upload.target`: `'filesystem'`
     - `ci.upload.outputDir`: `'./lighthouse-report'`

3. Create `e2e/lighthouse/run-lighthouse.ts`:
   - TypeScript script that:
     a. Checks if localhost:3000 is reachable (fetch with timeout); if not, prints instruction to start dev server and exits with code 1
     b. Runs `lhci autorun --config=e2e/lighthouse/lighthouse.config.ts` via child_process.execSync
     c. Prints path to generated HTML report
     d. Exits with the lhci exit code (0 = pass, non-zero = threshold violated)
   - This script does NOT auto-start Next.js -- dev must have it running (or use it in conjunction with emulators). This keeps it simple and fast.

4. Create `e2e/security/checks/audit-check.ts`:
   - Export `async function runAuditCheck(): Promise<CheckResult>`
   - Runs `npm audit --json --omit=dev --audit-level=high` via child_process
   - Also runs `npm --prefix functions audit --json --omit=dev --audit-level=high`
   - Parses JSON output, counts high/critical vulnerabilities
   - Returns `{ name: 'npm-audit', status: 'pass'|'warn'|'fail', details: string, highCount: number, criticalCount: number }`
   - `fail` if any critical, `warn` if any high, `pass` otherwise

5. Create `e2e/security/checks/header-check.ts`:
   - Export `async function runHeaderCheck(baseUrl: string): Promise<CheckResult>`
   - Fetches `baseUrl` and checks response headers:
     - `X-Content-Type-Options: nosniff` (expected from Next.js)
     - `X-Frame-Options` or `Content-Security-Policy frame-ancestors` present
     - `Strict-Transport-Security` present (only warn if missing, local dev won't have it)
   - Returns check result with details of which headers are present/missing

6. Create `e2e/security/checks/cors-check.ts`:
   - Export `async function runCorsCheck(baseUrl: string): Promise<CheckResult>`
   - Sends OPTIONS request to `baseUrl/api/backend/health` (or similar) with Origin header set to `http://evil-site.com`
   - Checks that `Access-Control-Allow-Origin` does NOT echo back the evil origin
   - Returns pass/fail result

7. Create `e2e/security/run-security-scan.ts`:
   - TypeScript script that:
     a. Imports and runs all check modules: audit-check, header-check, cors-check
     b. Accepts optional `--url` flag (default `http://localhost:3000`)
     c. Collects all results into a `SecurityReport` object
     d. Prints formatted console output with pass/warn/fail status per check
     e. Writes JSON report to `./security-report/security-scan.json`
     f. Exits with code 1 if any check is `fail`, 0 otherwise
   - Header and CORS checks are skipped with a warning if the URL is not reachable (allows running just npm audit without a server)

8. Add scripts to root `package.json`:
   - `"test:performance": "npx tsx e2e/lighthouse/run-lighthouse.ts"`
   - `"test:security": "npx tsx e2e/security/run-security-scan.ts"`

9. Install `tsx` as a dev dependency (for running TypeScript scripts directly without compilation).

10. Add to `.gitignore`: `lighthouse-report/`, `security-report/`
</steps>
