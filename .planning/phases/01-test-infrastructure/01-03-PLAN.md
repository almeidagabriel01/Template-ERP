# Plan 3: GitHub Actions CI Pipeline

## Goal
GitHub Actions automatically runs E2E, Lighthouse, and OWASP ZAP scans as parallel jobs on every PR, uploading test reports as downloadable artifacts and failing the PR on any suite failure.

## Context
Per D-04: GitHub Actions runs all 3 suites as parallel jobs on every PR, targeting ~10-12 min wall time. Each job uploads its report as a downloadable artifact. Per D-03: Full OWASP ZAP scan runs only in CI using the `zaproxy/zap-stable` Docker image (local uses the lightweight variant from Plan 2). This plan depends on Plans 1 and 2 because it references the scripts, configs, and directory structures they create.

1. Create `.github/workflows/` directory structure.

2. Create `.github/workflows/test-suite.yml`:

```yaml
name: Test Suite
on:
  pull_request:
    branches: [main, develop]

concurrency:
  group: tests-${{ github.head_ref }}
  cancel-in-progress: true

jobs:
  e2e:
    name: E2E Tests
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Install functions dependencies
        run: cd functions && npm ci

      - name: Build functions
        run: cd functions && npm run build

      - name: Install Playwright browsers
        run: npx playwright install chromium --with-deps

      - name: Install Firebase CLI
        run: npm install -g firebase-tools

      - name: Run E2E tests
        run: npm run test:e2e
        env:
          CI: true
          NEXT_PUBLIC_USE_FIREBASE_EMULATORS: 'true'
          NEXT_PUBLIC_FIREBASE_PROJECT_ID: demo-proops-test
          NEXT_PUBLIC_FIREBASE_API_KEY: demo-key
          NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN: demo-proops-test.firebaseapp.com
          NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET: demo-proops-test.appspot.com
          NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID: '000000000000'
          NEXT_PUBLIC_FIREBASE_APP_ID: '1:000000000000:web:demo'
          FIRESTORE_EMULATOR_HOST: '127.0.0.1:8080'
          FIREBASE_AUTH_EMULATOR_HOST: '127.0.0.1:9099'

      - name: Upload Playwright report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 14

      - name: Upload test results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-results
          path: test-results/
          retention-days: 14

  lighthouse:
    name: Lighthouse Performance
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build Next.js
        run: npm run build

      - name: Run Lighthouse CI
        run: |
          npm run build
          npx lhci autorun --config=e2e/lighthouse/lighthouse.config.ts
        env:
          LHCI_BUILD_CONTEXT__CURRENT_HASH: ${{ github.sha }}

      - name: Upload Lighthouse report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: lighthouse-report
          path: lighthouse-report/
          retention-days: 14

  security:
    name: Security Scan
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Install functions dependencies
        run: cd functions && npm ci

      - name: Run lightweight security checks
        run: npm run test:security -- --url=http://localhost:3000
        continue-on-error: true

      - name: Build Next.js for ZAP scan
        run: npm run build

      - name: Start Next.js production server
        run: npm start &
        env:
          PORT: 3000
          NEXT_PUBLIC_USE_FIREBASE_EMULATORS: 'true'
          NEXT_PUBLIC_FIREBASE_PROJECT_ID: demo-proops-test
          NEXT_PUBLIC_FIREBASE_API_KEY: demo-key
          NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN: demo-proops-test.firebaseapp.com
          NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET: demo-proops-test.appspot.com
          NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID: '000000000000'
          NEXT_PUBLIC_FIREBASE_APP_ID: '1:000000000000:web:demo'

      - name: Wait for server
        run: npx wait-on http://localhost:3000 --timeout 30000

      - name: Run OWASP ZAP baseline scan
        uses: zaproxy/action-baseline@v0.12.0
        with:
          target: 'http://localhost:3000'
          rules_file_name: '.zap-rules.tsv'
          cmd_options: '-a -j'
          allow_issue_writing: false

      - name: Upload ZAP report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: zap-report
          path: |
            zap-report.html
            zap-report.json
          retention-days: 14

      - name: Upload security scan report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: security-scan-report
          path: security-report/
          retention-days: 14
```

3. Create `.zap-rules.tsv` at project root -- a ZAP rules configuration file that sets thresholds:
   - Format: tab-separated with columns `rule_id`, `name`, `threshold`, `status`
   - Set common false-positive rules to WARN instead of FAIL (e.g., CSP missing on localhost, cookie flags on non-HTTPS)
   - Example entries:
     ```
     10038	Content Security Policy (CSP) Header Not Set	WARN	
     10049	Storable and Cacheable Content	WARN	
     10021	X-Content-Type-Options Header Missing	FAIL	
     ```

4. Refine the Lighthouse CI job to use `lhci` properly in CI mode:
   - The job should build Next.js first, then use `lhci autorun` which handles starting a local server from the build output
   - Override lighthouse config for CI: 3 runs instead of 1, collect on multiple URLs (login, dashboard if reachable)
   - The `lhci autorun` with `ci.upload.target: 'filesystem'` writes to `lighthouse-report/`

5. Ensure the E2E job handles the emulator lifecycle correctly:
   - The globalSetup in Plan 1 handles starting emulators, so CI just needs Firebase CLI installed and the right env vars
   - The `CI=true` env var is detected by playwright.config.ts to set retries=2

6. Add `.github/workflows/` to `.gitignore` exclusion (ensure it IS committed, not ignored).
