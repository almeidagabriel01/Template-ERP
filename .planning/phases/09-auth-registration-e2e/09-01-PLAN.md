<decisions>
D-01: Tests use unauthenticated { page } fixture (not authenticatedPage) — registration starts from a fresh, unauthenticated browser context.
D-02: Email uniqueness: `reg-test-${Date.now()}@test.com` per test — prevents collisions across runs.
D-03: Phone field (id="reg-phone") is left blank — it is optional and skipping it avoids Brazilian mobile phone format validation.
D-04: Niche select (id="niche") is left at its default value (automacao_residencial) — no interaction needed.
D-05: Step 3 (branding) has no required fields — skip all inputs and click "Finalizar" directly.
D-06: REG-02 reads Firestore via getTestDb() Admin SDK after signing in via signInWithEmailPassword() to get the UID. masterId is NOT verified (not set during self-signup).
D-07: REG-03 asserts the final URL is '/' — this proves auth state was established, session synced, and redirect happened for a free-plan user.
D-08: No afterEach cleanup — emulator resets per CI run; users from individual tests don't accumulate in CI.
D-09: waitForHomeRedirect() waits for URL to be exactly '/' (or starts with '/') with a 30s timeout to allow for the reload + auth provider read cycle.
</decisions>

<pitfalls>
P-01: The registration form is at /register, not /login — goto() navigates to "/register" directly. The app auto-sets mode="register" via pathname check.
P-02: After submitting step 3, there is a reload cycle (EmailVerificationPending → auto-proceed → reload → redirect). waitForHomeRedirect() must wait for the final URL '/' with a long timeout (30s) to survive this cycle.
P-03: The "Continuar" step navigation button may share the same text with other buttons — scope to the active step footer or use getByRole("button", { name: /continuar/i }).last() since inactive steps are hidden.
P-04: The "Finalizar" button on step 3 uses submitLabel="Finalizar" in StepNavigation — use getByRole("button", { name: /finalizar/i }).
P-05: After registration, signInWithEmailPassword() signs in as the newly registered user to get the UID (localId). This is a separate REST call to the auth emulator — it does NOT affect the browser page session.
P-06: getTestDb() requires the Firestore emulator to be running and FIRESTORE_EMULATOR_HOST to be set in the test environment. Playwright global setup handles this; no extra setup needed in the spec.
P-07: The users/{uid} document is created asynchronously during registration. Add a retry/poll loop when reading it (up to 5s) in case of a race between registration completion and Firestore write propagation in the emulator.
P-08: waitForVerificationScreen() is a helper method on the POM but may not need to be awaited explicitly in tests — the auto-proceed with SKIP_EMAIL_VERIFICATION means the screen appears and disappears quickly. It is provided for debugging purposes.
</pitfalls>

### Task 1: Create RegisterPage POM (`e2e/pages/register.page.ts`)

Create a new file `e2e/pages/register.page.ts` with a class-based POM following the pattern of `e2e/pages/login.page.ts`.

```typescript
import type { Page, Locator } from "@playwright/test";

export class RegisterPage {
  readonly page: Page;
  readonly nameInput: Locator;
  readonly emailInput: Locator;
  readonly passwordInput: Locator;
  readonly companyNameInput: Locator;

  constructor(page: Page) {
    this.page = page;
    this.nameInput = page.locator("#reg-name");
    this.emailInput = page.locator("#email");
    this.passwordInput = page.locator("#password");
    this.companyNameInput = page.locator("#companyName");
  }

  async goto(): Promise<void> {
    await this.page.goto("/register");
  }

  async isLoaded(): Promise<boolean> {
    await this.page.waitForURL(/\/register/, { timeout: 15000 });
    await this.nameInput.waitFor({ state: "visible", timeout: 15000 });
    return true;
  }

  async fillStep1(data: { name: string; email: string; password: string }): Promise<void> {
    await this.nameInput.fill(data.name);
    await this.emailInput.fill(data.email);
    await this.passwordInput.fill(data.password);
    // Click "Continuar" to advance to step 2
    await this.page.getByRole("button", { name: /continuar/i }).click();
    // Wait for step 2 to appear (companyName input visible)
    await this.companyNameInput.waitFor({ state: "visible", timeout: 10000 });
  }

  async fillStep2(data: { companyName: string }): Promise<void> {
    await this.companyNameInput.fill(data.companyName);
    // Niche select left at default — no interaction
    // Click "Continuar" to advance to step 3
    await this.page.getByRole("button", { name: /continuar/i }).click();
    // Wait for step 3 — the "Finalizar" button
    await this.page.getByRole("button", { name: /finalizar/i }).waitFor({ state: "visible", timeout: 10000 });
  }

  async submitStep3(): Promise<void> {
    // Step 3 has only optional branding fields — click Finalizar directly
    await this.page.getByRole("button", { name: /finalizar/i }).click();
  }

  async waitForVerificationScreen(): Promise<void> {
    // The EmailVerificationPending screen appears briefly before auto-proceeding
    // With NEXT_PUBLIC_SKIP_EMAIL_VERIFICATION=true, this resolves automatically
    await this.page.waitForFunction(
      () => document.body.innerText.includes("verificação") || document.body.innerText.includes("email"),
      { timeout: 15000 },
    ).catch(() => {
      // Screen may have appeared and disappeared too quickly — that's fine
    });
  }

  async waitForHomeRedirect(): Promise<void> {
    // After registration: EmailVerificationPending → auto-proceed → reload → redirect to '/'
    // The reload cycle can take up to 15s — use a 30s timeout
    await this.page.waitForURL(/^\/$|^\/[^l]/, { timeout: 30000 });
  }
}
```

### Task 2: Create registration spec (`e2e/auth/auth-registration.spec.ts`)

```typescript
/**
 * Auth Registration E2E tests — REG-01 through REG-03.
 *
 * REG-01: New user completes the 3-step registration form and submits successfully
 * REG-02: Newly registered tenant has correct Firestore documents (users/{uid}, tenants/{tenantId})
 * REG-03: New tenant lands on '/' after completing registration (free-plan redirect)
 */

import { test, expect } from "../fixtures/auth.fixture";
import { RegisterPage } from "../pages/register.page";
import { signInWithEmailPassword } from "../helpers/firebase-auth-api";
import { getTestDb } from "../helpers/admin-firestore";

test.describe("Auth Registration", () => {
  // REG-01: Form submission
  test("REG-01: completes the registration form and submits successfully", async ({ page }) => {
    const registerPage = new RegisterPage(page);
    const timestamp = Date.now();
    const testEmail = `reg-test-${timestamp}@test.com`;
    const testPassword = "TestReg1234!";
    const testName = `Teste Registro ${timestamp}`;
    const testCompanyName = `Empresa Teste ${timestamp}`;

    await registerPage.goto();
    await registerPage.isLoaded();

    await registerPage.fillStep1({ name: testName, email: testEmail, password: testPassword });
    await registerPage.fillStep2({ companyName: testCompanyName });
    await registerPage.submitStep3();

    // Wait for the home redirect — proves registration completed and session was established
    await registerPage.waitForHomeRedirect();

    // Final URL must be '/'
    expect(page.url()).toMatch(/\/$/);
  });

  // REG-02: Firestore provisioning
  test("REG-02: newly registered tenant has correct Firestore documents", async ({ page }) => {
    const registerPage = new RegisterPage(page);
    const timestamp = Date.now();
    const testEmail = `reg-test-${timestamp}@test.com`;
    const testPassword = "TestReg1234!";
    const testName = `Teste Claims ${timestamp}`;
    const testCompanyName = `Empresa Claims ${timestamp}`;

    await registerPage.goto();
    await registerPage.isLoaded();

    await registerPage.fillStep1({ name: testName, email: testEmail, password: testPassword });
    await registerPage.fillStep2({ companyName: testCompanyName });
    await registerPage.submitStep3();

    // Wait for registration to complete (home redirect)
    await registerPage.waitForHomeRedirect();

    // Get the UID by signing in via the Auth emulator REST API
    const { localId: uid } = await signInWithEmailPassword(testEmail, testPassword);
    expect(uid).toBeTruthy();

    const expectedTenantId = `tenant_${uid}`;
    const db = getTestDb();

    // Poll for users/{uid} doc with up to 5s retry (emulator write may lag slightly)
    let userDoc: FirebaseFirestore.DocumentSnapshot | null = null;
    for (let i = 0; i < 10; i++) {
      userDoc = await db.collection("users").doc(uid).get();
      if (userDoc.exists) break;
      await new Promise((resolve) => setTimeout(resolve, 500));
    }

    expect(userDoc?.exists).toBe(true);
    const userData = userDoc!.data()!;
    expect(userData["tenantId"]).toBe(expectedTenantId);
    expect(userData["role"]).toBe("free");
    expect(userData["email"]).toBe(testEmail);

    // Verify tenants/{tenantId} doc
    const tenantDoc = await db.collection("tenants").doc(expectedTenantId).get();
    expect(tenantDoc.exists).toBe(true);
    const tenantData = tenantDoc.data()!;
    expect(tenantData["name"]).toBe(testCompanyName);
  });

  // REG-03: Dashboard access
  test("REG-03: new tenant lands on '/' after registration (free-plan redirect)", async ({ page }) => {
    const registerPage = new RegisterPage(page);
    const timestamp = Date.now();
    const testEmail = `reg-test-${timestamp}@test.com`;
    const testPassword = "TestReg1234!";
    const testName = `Teste Dashboard ${timestamp}`;
    const testCompanyName = `Empresa Dashboard ${timestamp}`;

    await registerPage.goto();
    await registerPage.isLoaded();

    await registerPage.fillStep1({ name: testName, email: testEmail, password: testPassword });
    await registerPage.fillStep2({ companyName: testCompanyName });
    await registerPage.submitStep3();

    // Wait for the home redirect — proves auth flow completed and free-plan redirect happened
    await registerPage.waitForHomeRedirect();

    // Free-plan users are redirected to '/' (handleRedirectAfterAuth role check in useLoginForm)
    expect(page.url()).toMatch(/\/$/);

    // The page should not be the login or register page — user is authenticated
    expect(page.url()).not.toContain("/login");
    expect(page.url()).not.toContain("/register");
  });
});
```
