
2. Replace the `sessionId` useState initializer. Current code (lines 78-83):
   ```typescript
   const [sessionId, setSessionId] = useState<string>(() => {
     // Only restore from localStorage on client, only for persistent plans
     if (typeof window === "undefined") return generateSessionId();
     if (!persistHistory || !tenantId) return generateSessionId();
     return localStorage.getItem(getStorageKey(tenantId)) ?? generateSessionId();
   });
   ```

   Replace with:
   ```typescript
   const [sessionId, setSessionId] = useState<string>(generateSessionId);
   ```

3. Add `isRestoredRef` after the sessionId state declaration:
   ```typescript
   const isRestoredRef = useRef(false);
   ```

4. Add the restoration useEffect BEFORE the existing persist useEffect (before line 89). This effect runs once when persistHistory and tenantId become available, restoring the stored session ID if one exists:
   ```typescript
   // Restore sessionId from localStorage once auth context is ready (Pro/Enterprise only)
   useEffect(() => {
     if (isRestoredRef.current) return;
     if (!persistHistory || !tenantId) return;
     isRestoredRef.current = true;
     const stored = localStorage.getItem(getStorageKey(tenantId));
     if (stored) {
       setSessionId(stored);
     }
   }, [persistHistory, tenantId]);
   ```

5. Modify the existing persist useEffect (currently lines 89-92) to guard against running before restoration:
   Current:
   ```typescript
   useEffect(() => {
     if (!persistHistory || !tenantId) return;
     localStorage.setItem(getStorageKey(tenantId), sessionId);
   }, [sessionId, tenantId, persistHistory]);
   ```

   Change to:
   ```typescript
   useEffect(() => {
     if (!persistHistory || !tenantId || !isRestoredRef.current) return;
     localStorage.setItem(getStorageKey(tenantId), sessionId);
   }, [sessionId, tenantId, persistHistory]);
   ```

   The only change is adding `|| !isRestoredRef.current` to the guard. This prevents the persist effect from saving the randomly-generated sessionId before the restoration effect has a chance to run.

Do NOT modify the loadHistory useEffect, startNewSession, loadSession, or the return object.
