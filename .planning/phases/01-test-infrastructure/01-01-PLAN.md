# Plan 1: Playwright + Firebase Emulators + Seed Data + Page Object Model

## Goal
Developer runs `npm run test:e2e` and a full Playwright test suite executes against auto-started Firebase Emulators with deterministic seed data -- zero manual setup.

## Context
This is the foundation of the entire testing milestone. Phase 1 builds the test infrastructure that all subsequent phases depend on. There are currently zero tests in the project. This plan establishes Playwright with TypeScript, auto-starting Firebase Emulators via globalSetup, a deterministic seed data factory, Page Object Model classes for main pages, and reusable auth fixtures. A single smoke test validates the full pipeline works end-to-end.

Per D-01: Firebase Emulators auto-start via Playwright globalSetup -- no manual steps.
Per D-02: Seed factory runs once per test run in globalSetup, creating 2 tenants with users, proposals, transactions, wallets.

## Tasks

<task id="1.1.1" title="Install Playwright, configure with globalSetup for Firebase Emulators">
<read_first>
- package.json -- current scripts and dependencies to understand naming conventions
- firebase.json -- emulator ports (Auth:9099, Firestore:8080, Functions:5001, Storage:9199)
- tsconfig.json -- current TypeScript config to ensure e2e tsconfig aligns
</read_first>
<steps>
1. Install dev dependencies: `@playwright/test` and `firebase-tools` (if not already global). Run `npx playwright install chromium` to get the browser binary.

2. Create `e2e/tsconfig.json` extending the root tsconfig but with `module: "commonjs"` (Playwright runs in Node), `outDir: "./dist"`, `rootDir: "."`, and including `**/*.ts`. Add path alias `@e2e/*` pointing to `./*`.

3. Create `playwright.config.ts` at project root:
   - `testDir: './e2e'` with `testMatch: '**/*.spec.ts'`
   - `globalSetup: './e2e/global-setup.ts'`
   - `globalTeardown: './e2e/global-teardown.ts'`
   - `use.baseURL: 'http://localhost:3000'` (Next.js dev server)
   - `webServer` config: command `npm run dev`, port 3000, reuseExistingServer true, env vars `NEXT_PUBLIC_USE_FIREBASE_EMULATORS=true` plus all `NEXT_PUBLIC_FIREBASE_*` vars pointing to emulator project
   - `projects`: single chromium project for now
   - `reporter`: `[['html', { open: 'never' }], ['list']]`
   - `retries: 0` for local, `retries: 2` for CI (detect via `process.env.CI`)
   - `timeout: 30000`, `expect.timeout: 10000`
   - Set env vars in `use`: `FIRESTORE_EMULATOR_HOST=127.0.0.1:8080`, `FIREBASE_AUTH_EMULATOR_HOST=127.0.0.1:9099`

4. Create `e2e/global-setup.ts`:
   - Start Firebase Emulators as a child process using `firebase emulators:start --project demo-proops-test --import=./e2e/seed/emulator-data --only auth,firestore,storage` (using demo project prefix avoids needing real Firebase credentials)
   - Wait for emulators to be ready by polling `http://127.0.0.1:4400/emulators` (Emulator Hub) with fetch, retrying every 1s up to 30s
   - Once emulators ready, import and call `seedAll()` from `e2e/seed/seed-factory.ts`
   - Store the child process reference in a global file (e.g., write PID to `.emulator-pid`) for teardown
   - Export the setup function as default

5. Create `e2e/global-teardown.ts`:
   - Read the emulator PID from `.emulator-pid`
   - Kill the process tree (use `process.kill` or `tree-kill` package)
   - Delete the PID file
   - Export as default

6. Create `.env.test` at project root with:
   ```
   NEXT_PUBLIC_USE_FIREBASE_EMULATORS=true
   NEXT_PUBLIC_FIREBASE_API_KEY=demo-key
   NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=demo-proops-test.firebaseapp.com
   NEXT_PUBLIC_FIREBASE_PROJECT_ID=demo-proops-test
   NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=demo-proops-test.appspot.com
   NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=000000000000
   NEXT_PUBLIC_FIREBASE_APP_ID=1:000000000000:web:demo
   FIRESTORE_EMULATOR_HOST=127.0.0.1:8080
   FIREBASE_AUTH_EMULATOR_HOST=127.0.0.1:9099
   FIREBASE_STORAGE_EMULATOR_HOST=127.0.0.1:9199
   ```

7. Add scripts to root `package.json`:
   - `"test:e2e": "npx playwright test"`
   - `"test:e2e:ui": "npx playwright test --ui"`
   - `"test:e2e:debug": "npx playwright test --debug"`

8. Add to `.gitignore`: `playwright-report/`, `test-results/`, `.emulator-pid`, `e2e/dist/`

9. Create `e2e/smoke.spec.ts` -- a minimal smoke test:
   - Import `test` and `expect` from `@playwright/test`
   - Test "app loads login page": navigate to `/`, expect to see login form or redirect to login route
   - This validates the full pipeline: emulators running, Next.js serving, Playwright executing
</steps>

<task id="1.1.2" title="Create seed data factory and Page Object Model classes with auth fixture">
<read_first>
- e2e/global-setup.ts -- to understand how seedAll is called (created in previous task)
- firebase.json -- emulator ports for Admin SDK connection
- src/types/ -- browse existing TypeScript interfaces for proposals, transactions, wallets to align seed data shapes
- src/app/(main)/proposals/ -- understand proposal routes for POM
- src/app/(main)/transactions/ -- understand transaction routes for POM
- src/app/login/ -- understand login page structure for LoginPage POM (or equivalent auth route)
</read_first>
<steps>
1. Create `e2e/seed/seed-factory.ts`:
   - Initialize Firebase Admin SDK with `{ projectId: 'demo-proops-test' }` connecting to emulators via env vars `FIRESTORE_EMULATOR_HOST` and `FIREBASE_AUTH_EMULATOR_HOST`
   - Export `async function seedAll()` that calls tenant, user, proposal, transaction, and wallet seeders in order
   - Export `async function clearAll()` that deletes all seeded data (for potential per-suite reset)
   - Use deterministic IDs (e.g., `tenant-alpha`, `tenant-beta`, `user-admin-alpha`, `user-member-alpha`) so tests can reference them without querying

2. Create `e2e/seed/data/tenants.ts`:
   - Export `TENANT_ALPHA` and `TENANT_BETA` objects with `{ id: 'tenant-alpha', name: 'Alpha Corp', niche: 'automacao_residencial', ... }` and `{ id: 'tenant-beta', name: 'Beta Ltd', niche: 'cortinas', ... }`
   - Export `async function seedTenants(db: Firestore)` that writes both tenant docs to `tenants/{id}`

3. Create `e2e/seed/data/users.ts`:
   - Define 4 users: admin-alpha (role: master), member-alpha (role: member), admin-beta (role: master), member-beta (role: member)
   - Each user has deterministic email (`admin@alpha.test`), password (`Test1234!`), uid, and custom claims (`tenantId`, `role`, `masterId`)
   - Export `async function seedUsers(auth: Auth, db: Firestore)` that creates users in Firebase Auth emulator via Admin SDK (`auth.createUser`), sets custom claims (`auth.setCustomUserClaims`), and writes user docs to `users/{uid}` in Firestore
   - Export user constants (USER_ADMIN_ALPHA, USER_MEMBER_ALPHA, etc.) for test reference

4. Create `e2e/seed/data/proposals.ts`:
   - Create 3 proposals for tenant-alpha (draft, sent, approved statuses) and 1 for tenant-beta
   - Use deterministic IDs (`proposal-alpha-draft`, `proposal-alpha-sent`, etc.)
   - Include realistic fields matching the project's proposal schema (title, status, items array with at least 1 item, contact reference, dates)
   - Export `async function seedProposals(db: Firestore)`

5. Create `e2e/seed/data/transactions.ts`:
   - Create 3 transactions for tenant-alpha (income-paid, expense-pending, installment) and 1 for tenant-beta
   - Deterministic IDs (`transaction-alpha-income`, etc.)
   - Include realistic fields: type (income/expense), status (paid/pending), amount, wallet reference, dates
   - Export `async function seedTransactions(db: Firestore)`

6. Create `e2e/seed/data/wallets.ts`:
   - Create 2 wallets for tenant-alpha (`wallet-alpha-main`, `wallet-alpha-savings`) and 1 for tenant-beta
   - Include balance field (denormalized per project architecture)
   - Export `async function seedWallets(db: Firestore)`

7. Create `e2e/pages/login.page.ts`:
   - Export `class LoginPage` with constructor taking Playwright `Page`
   - Methods: `goto()`, `fillEmail(email)`, `fillPassword(password)`, `submit()`, `login(email, password)` (combines fill+submit), `getErrorMessage()`
   - Use resilient selectors: prefer `data-testid` attributes, fall back to role-based selectors (`getByRole('button', { name: /entrar|login/i })`)
   - Note: Actual selectors will need adjustment based on real DOM -- use best-guess selectors with clear comments

8. Create `e2e/pages/dashboard.page.ts`:
   - Export `class DashboardPage` with `goto()`, `isLoaded()` (checks for dashboard-specific element), `getWelcomeText()`, `navigateTo(section)` methods

9. Create `e2e/pages/proposals.page.ts`:
   - Export `class ProposalsPage` with `goto()`, `isLoaded()`, `getProposalCount()`, `clickNewProposal()`, `getProposalByTitle(title)` methods

10. Create `e2e/pages/transactions.page.ts`:
    - Export `class TransactionsPage` with `goto()`, `isLoaded()`, `getTransactionCount()`, `clickNewTransaction()` methods

11. Create `e2e/fixtures/base.fixture.ts`:
    - Extend Playwright's base `test` to export a custom `test` with typed fixtures
    - Provide page object instances: `loginPage`, `dashboardPage`, `proposalsPage`, `transactionsPage`
    - Each fixture instantiates the POM class with the current `page`

12. Create `e2e/fixtures/auth.fixture.ts`:
    - Extend the base fixture to add an `authenticatedPage` fixture
    - This fixture uses `LoginPage` to log in as `USER_ADMIN_ALPHA` (email: `admin@alpha.test`, password: `Test1234!`)
    - After login, waits for navigation to dashboard (or authenticated route)
    - Provides the authenticated `page` to tests
    - Also export `authenticatedAsBeta` fixture for tenant B tests (for future cross-tenant isolation tests)

13. Update `e2e/smoke.spec.ts` to use the auth fixture:
    - Add a second test: "authenticated user sees dashboard" using `authenticatedPage` fixture
    - Verify dashboard loads after login
</steps>
